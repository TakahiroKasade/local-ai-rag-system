# 🏟️ 本地專屬 AI 知識庫 (Local RAG System)

這是一個基於 **Ollama** 與 **ChromaDB** 打造的本地端檢索增強生成 (RAG) 系統。本專案允許使用者將私有的文件（PDF, Word, Markdown, TXT）匯入本地向量資料庫，並透過網頁介面與 AI 進行安全、私密且完全本地化的對話。

---

## 🛠️ 使用技術

*   **核心模型**: [Ollama](https://ollama.com/) - 運行 Llama 3.2 (本地大型語言模型)。
*   **網頁介面**: [Streamlit](https://streamlit.io/) - 提供流暢的聊天互動視窗。
*   **向量資料庫**: [ChromaDB](https://www.trychroma.com/) - 儲存文件特徵並進行高效檢索。
*   **開發語言**: Python 3.12+
*   **文件解析**: 
    *   `pypdf`: 解析 PDF 檔案。
    *   `python-docx`: 解析 Word (.docx) 檔案。
    *   原生支援 Markdown (.md) 與純文字 (.txt)。

---

## ✨ 專案特點

1.  **完全本地化**: 資料不需上傳雲端，保護企業或個人隱私。
2.  **多格式支援**: 一鍵匯入 PDF、Word、Markdown 與 TXT 檔案。
3.  **智慧切片 (Chunking)**: 自動將長文件切成 500 字的小塊，優化 AI 讀取速度與精確度。
4.  **串流回覆 (Streaming)**: 回答即時跳出，無需等待整段生成。
5.  **智慧過濾**: 提問時若提到特定檔名，系統會自動鎖定該檔案進行深度分析。

## 🚀 操作指南

### 1. 環境準備
確保您的電腦已安裝 Ollama 並下載模型：
```bash
ollama run llama3.2
```

### 2. 資料入庫
將您的文件放入 `my_data/` 資料夾中，然後執行入庫腳本將資料向量化：
```bash
python ingest_folder.py
```
*註：每次新增檔案或修改內容後，皆需執行此步驟。*

### 3. 啟動網頁聊天室
執行以下指令開啟瀏覽器介面：
```bash
streamlit run app.py
```

### 4. 輔助工具
*   **檢查資料庫狀態**: `python check_db.py` (查看目前存入了哪些片段)。
*   **CLI 測試模式**: `python ollama_query.py` (不啟動網頁，直接在終端機對話)。

---

## 📂 目錄結構說明

*   `app.py`: Streamlit 網頁主程式。
*   `ingest_folder.py`: 負責讀取檔案、切片並存入資料庫。
*   `check_db.py`: 資料庫內容掃描工具。
*   `my_data/`: 存放原始文件的資料夾。
*   `chroma_db/`: 存放向量化資料的持久化目錄。
*   `ollama_query.py`: 本地端對話測試腳本 (CLI)。

## ⚠️ 注意事項

*   **硬體效能**: 本專案運行於本地 CPU/GPU，回應速度取決於電腦效能。若感官較慢，可考慮調降檢索片段數量。
*   **檔案限制**: 建議單個 PDF 不要超過 100 頁，以獲得最佳的檢索品質。
